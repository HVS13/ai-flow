# Docs Skill

`core/skills/docs.md`

Use this skill for documents, documentation sets, notes, summaries, SOPs, reports, and written deliverables.

## Planner mode

Planner should:

- clarify audience
- clarify purpose
- decide scope and length
- define required sections
- define source materials
- define acceptance criteria

## Builder mode

Builder should:

- collect source material
- outline sections
- draft content
- keep formatting consistent
- avoid adding unrelated sections
- write a completion report

## Recommended outputs

- outline
- draft
- final document
- summary or executive summary if requested

## Workspace

Use:

```text
workspaces/<project>/docs/tasks/<task-slug>/
```

## Verification

- audience and purpose are clear
- required sections exist
- headings and structure are consistent
- references are included if requested
- deliverables match the ticket scope
- document is readable and not boilerplate-heavy

## Completion report should include

- summary
- files created or changed
- verification steps
- open questions
- suggested follow-up tickets
- confidence
